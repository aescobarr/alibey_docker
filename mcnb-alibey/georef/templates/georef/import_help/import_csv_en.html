The accepted formats are:

  • EXCEL: Microsoft Excel file

  • CSV: The file must be a valid CSV, using the ';' symbol as the column separator. It is assumed that the file contains a header row with column names, so the first line of the file is never processed. To avoid issues with special characters, the file should be encoded in UTF-8.

Below is a list of the columns that the file must contain. Mandatory columns are shown in red.

  • Name
    • String with the name of the site name.
  • Type
    • Type of site name. A value present in the lookup table of site name types must be indicated.
  • Aquatic
    • Indicates whether the site name is aquatic or terrestrial. It is considered aquatic if this field contains any of the values ['true' | 'cert' | 's√≠' | 'si' | '1' | 'cierto' | 'verdadero'], and terrestrial otherwise.
  • Parent node
    • Full name of the site name in which the current site name is contained. The site name name must be in the format: [name] - [country] ([site name_type]) (Aquatic|Terrestrial).
    • [name] can be any text string, for example "Calonge de Segarra". It cannot be left blank.
    • [country] is optional. Any country name may be indicated here, as long as it is present in the Countries lookup table. Otherwise, leave it blank.
    • [site name_type] is mandatory. Any value from the site name types lookup table may be used here.
    • Finally, it must be indicated whether the site name is aquatic or terrestrial.
  • Version number
    • The version number of the current site name. It must be a valid integer.
  • Version qualifier
    • The version qualifier of the current site name. It can be any valid value from the Version Qualifiers lookup table.
  • Captured version of the resource
    • Name of the georeferencing resource used for geolocating the site name. It can be any name from the list of Resources.
  • site name name in the resource
    • Name of the site name in the georeferencing resource used. It may differ from the site name name in Alibey; this field is optional.
  • Date
    • Date of georeferencing. A wide range of formats is accepted, but it's recommended to use DD/MM/YYYY (e.g., 03/05/2009). This field is optional; leave it blank if unspecified.
  • Original x coordinate
    • Original x coordinate of the site name, in the original units (either meters or latitude/longitude)
  • Original y coordinate
    • Original y coordinate of the site name, in the original units (either meters or latitude/longitude)
  • Maximum depth altitude (m)
    • Maximum depth, in meters. Decimals are allowed. This field is not mandatory.
  • Minimum depth altitude (m)
    • Minimum depth, in meters. Decimals are allowed. This field is not mandatory.
  • Coordinate uncertainty (m)
    • Coordinate uncertainty in meters around the original coordinate. This field is mandatory.
  • Georeferencer
    • Name of the georeferencer, in the form [first name last name].
  • Observations
    • Observations from the georeferencing process. Not mandatory.

When uploading the file, all its rows will be reviewed to ensure there are no errors. If there are any, a window will appear listing them along with the problematic line and column. The user must resolve all these errors before loading the data. Once loaded, an output report will be displayed with a list of the records that have been added. The output report can be downloaded in CSV format.
