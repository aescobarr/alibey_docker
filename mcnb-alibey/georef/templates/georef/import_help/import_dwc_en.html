The import in Darwin Core format supports the following formats:

  • EXCEL: Microsoft Excel file

  • CSV: The file must be a valid CSV, using the ';' symbol as the column separator. It is assumed that the file has a first row with column names, so the first line of the file is never processed. To avoid issues with special characters, the file should be UTF-8 encoded.

Below is a list of columns that the file must contain. The order is not important, but the column name is. The correct column name is the one shown in parentheses. Required columns are shown in red; semi-required columns are shown in orange depending on the specific site (see field description).

  • Place name (locality).
    • The name of the place must be in the form: [name] - [country] ([site_type]) (Aquàtic|Terrestre).
    • The [name] can be any text string, for example "Calonge de Segarra". It cannot be blank.
    • The [country] is optional. You can specify the name of any country, which must be present in the Country thesaurus. If not, leave it blank.
    • The [site_type] is required. You can specify any value from the site type thesaurus.
    • Finally, indicate whether the place is Aquàtic or Terrestre.
  • Higher geographies (higherGeography)
    • This field is a lineage string of the place name. It is a list of place names separated by the '|' character. The rightmost place name is the one immediately above the one being imported.
    • For example, if importing the place "Calonge de Segarra", the string would be: "Europa - (continent) (Terrestre) | Espanya - Espanya (estat) (Terrestre) | Catalunya - Espanya (comunitat autònoma) (Terrestre) | Lleida - Catalunya (província) (Terrestre) | Urgell - Espanya (comarca) (Terrestre)", indicating that "Calonge de Segarra" is located in Anoia.
  • Version captured from resource (georeferenceSources)
    • Name of the georeferencing resource used. It must be present in the Resources list.
  • Original x coordinate (verbatimLongitude)
    • Original x coordinate of the place, in original units (either meters or lat/long).
  • Original y coordinate (verbatimLatitude)
    • Original y coordinate of the place, in original units (either meters or lat/long).
  • Date (georeferencedDate)
    • Date of the georeferencing. A wide range of formats is accepted, but DD/MM/YYYY is recommended (e.g., 03/05/2009). If not specified, the import date is used.
  • Georeferencer (georeferencedBy)
    • Name of the georeferencer, in the form [first name, last name].
  • Maximum depth height(m) (maximumDepthInMeters)
    • Maximum depth, in meters. Decimals are allowed. This field is not mandatory.
  • Minimum depth height(m) (minimumDepthInMeters)
    • Minimum depth, in meters. Decimals are allowed. This field is not mandatory.
  • Observations (georeferenceRemarks)
    • Comments field. Not mandatory.
  • Coordinate uncertainty (m) (coordinateUncertaintyInMeters)
      Coordinate uncertainty in meters around the original coordinate. This field is only mandatory if the geometry is a point. Otherwise, it is calculated based on the geometry's uncertainty radius.
  • Place geometry (footprintWKT)
      Geometry in WKT (well-known text) format of the place. This geometry will be used to calculate the centroid of the place and coordinate uncertainty.
  • Geometry reference system (footprintSRS)
    • Identifier with the EPSG code of the reference system (e.g., "epsg:4326")

When loading the file, all rows are checked to ensure there are no errors. If any are found, a window will appear with a list of them, indicating the problematic line and column. The user must resolve all errors before loading the data. Once the data is loaded, an output report will be displayed with a list of the added records. The output report can be downloaded in CSV format.
